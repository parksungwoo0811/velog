<thead>
<tr>
<th>용어</th>
<th>설명</th>
</tr>
</thead>

<h1 id="서비스-출시-·-운영-용어">서비스 출시 · 운영 용어</h1>
<table>
<thead>
<tr>
<th>용어</th>
<th>설명</th>
</tr>
</thead>
<tbody><tr>
<td><strong>MVP (Minimum Viable Product)</strong></td>

<tr>
<td><strong>Release</strong></td>

</tr>
<tr>
<td><strong>Launch</strong></td>
<td>새로운 서비스나 제품을 공식적으로 공개하는 것</td>
</tr>
<tr>
<td><strong>Go Live</strong></td>
<td>실제 운영 환경에서 서비스가 시작되는 시점</td>
</tr>
<tr>
<td><strong>Hotfix</strong></td>
<td>긴급 버그를 즉시 수정하여 배포하는 작업</td>
</tr>
<tr>
<td><strong>Deployment</strong></td>
<td>개발한 코드를 서버나 클라우드 환경에 배포하는 과정</td>
</tr>
<tr>
<td><strong>Rollout</strong></td>
<td>기능을 모든 사용자에게 한 번에 공개하지 않고 단계적으로 공개하는 방식</td>
</tr>
<tr>
<td><strong>Rolling Update</strong></td>
<td>서버를 순차적으로 업데이트하여 서비스 중단 없이 배포하는 방식</td>
</tr>
<tr>
<td><strong>Beta Release</strong></td>
<td>정식 출시 전 테스트를 위한 공개 버전</td>
</tr>
<tr>
<td><strong>Rollback</strong></td>
<td>문제가 발생했을 때 이전 안정 버전으로 되돌리는 작업</td>
</tr>
<tr>
<td><strong>Patch</strong></td>
<td>버그 수정이나 보안 개선을 위한 작은 업데이트</td>
</tr>
<tr>
<td><strong>Versioning</strong></td>
<td>서비스나 API 버전을 관리하는 것</td>
</tr>
<tr>
<td><strong>Issue</strong></td>
<td>해결해야 할 작업이나 버그를 관리하는 단위(티켓)</td>
</tr>
<tr>
<td><strong>Roadmap</strong></td>
<td>제품의 장기적인 방향과 계획을 정리한 전략 문서</td>
</tr>
</tbody></table>

<h1 id="한눈에-비교">한눈에 비교</h1>
<table>
<thead>
<tr>
<th>비교</th>
<th>차이점</th>
</tr>
</thead>
<tbody><tr>
<td><strong>Wireframe ↔ Mockup ↔ Prototype</strong></td>
<td>구조 설계 → 디자인 적용 → 실제 동작 구현</td>
</tr>
<tr>
<td><strong>UI ↔ UX</strong></td>
<td>UI는 화면, UX는 사용 경험</td>
</tr>
<tr>
<td><strong>Release ↔ Launch</strong></td>
<td>Release는 버전 배포, Launch는 서비스 공개</td>
</tr>
<tr>
<td><strong>Deployment ↔ Rollout</strong></td>
<td>Deployment는 배포, Rollout은 사용자에게 점진적 공개</td>
</tr>
<tr>
<td><strong>Patch ↔ Hotfix</strong></td>
<td>Patch는 일반 수정, Hotfix는 긴급 수정</td>
</tr>
<tr>
<td><strong>Logging ↔ Monitoring</strong></td>
<td>Logging은 기록, Monitoring은 실시간 감시</td>
</tr>
<tr>
<td><strong>Backup ↔ Restore</strong></td>
<td>Backup은 저장, Restore는 복구</td>
</tr>
<tr>
<td><strong>VM ↔ Container</strong></td>
<td>VM은 운영체제까지 가상화, Container는 실행 환경만 격리</td>
</tr>
<tr>
<td><strong>Hybrid Cloud ↔ Multi Cloud</strong></td>
<td>Hybrid는 온프레미스+클라우드, Multi는 여러 클라우드 사용</td>
</tr>
<tr>
<td><strong>Product Backlog ↔ Sprint Backlog</strong></td>
<td>전체 작업 목록 ↔ 이번 Sprint 작업 목록</td>
</tr>
</tbody></table>

<h1 id="핵심-암기-포인트">핵심 암기 포인트</h1>
<ul>
<li><strong>Wireframe → Mockup → Prototype</strong> 순으로 완성도가 높아진다.</li>
<li><strong>UI는 화면</strong>, <strong>UX는 경험</strong>이다.</li>
<li><strong>MVP</strong>는 최소 기능 제품이다.</li>
<li><strong>Deployment</strong>는 배포, <strong>Launch</strong>는 공개이다.</li>
<li><strong>Logging</strong>은 기록, <strong>Monitoring</strong>은 감시이다.</li>
<li><strong>Backup</strong>은 저장, <strong>Restore</strong>는 복구이다.</li>
<li><strong>Container + Docker + Kubernetes</strong>는 함께 자주 등장한다.</li>
<li><strong>CI/CD</strong>는 코드 작성부터 배포까지 자동화하는 개발 파이프라인이다.</li>
<li><strong>Scrum</strong>은 Agile을 실천하는 대표적인 프레임워크이다.</li>
</ul>
